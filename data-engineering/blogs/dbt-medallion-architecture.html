dbt Medallion Architecture

Building scalable, maintainable data warehouses with dbt's layered approach — Bronze, Silver, Gold

June 2024 • dbt, Databricks, Data Modeling

What is Medallion Architecture?

The Medallion Architecture (Bronze-Silver-Gold) is a design pattern for organizing data in a lakehouse. Each layer has a specific purpose: raw ingestion, transformation, and business-ready analytics. When implemented with dbt, it becomes a powerful, maintainable system.

Bronze Layer (Raw Data)

Stores raw, unmodified data exactly as received from source systems. Think of it as your data landing zone.

  • Direct copy of source data (COPY INTO or ELT)
  • Minimal schema inference
  • Immutable append-only storage
  • No transformations or validations

Silver Layer (Cleaned & Deduplicated)

dbt models run here to clean, deduplicate, and validate data. Business logic begins to emerge.

Gold Layer (Business Analytics)

Aggregate tables optimized for specific use cases. BI tools query directly from here.

  • Pre-aggregated metrics and KPIs
  • Denormalized for fast query performance
  • Domain-specific marts (finance, marketing, operations)
  • Materialized as tables (not views) for speed

Real-World Implementation

In a recent project, I implemented medallion architecture for a fintech company processing 500M transactions daily:

  • Bronze: 50GB daily raw transaction data from 12 source systems
  • Silver: Standardized schema, deduplication, lineage tracking (12GB after cleanup)
  • Gold: 4 analytics marts (daily balance, transaction summary, fraud scoring, reporting)

Using dbt's incremental models and staging tables, the entire daily pipeline ran in 18 minutes. Without medallion + dbt, it took 3 hours with full reprocessing.

dbt Best Practices

  • Staging models: Rename columns, fix data types at Bronze→Silver boundary
  • Source freshness: Track when source data was last updated
  • Tests: NOT NULL, unique, referential integrity at every layer
  • Documentation: Describe business meaning, not just column names
  • Macros: Abstract common transformations (date standardization, string cleaning)

Key Takeaways

  • Medallion keeps data organized. Debugging becomes easier when layers have clear boundaries
  • dbt's dependency tracking ensures correct execution order. No manual orchestration needed
  • Incremental models reduce data movement. Silver and Gold scale efficiently
  • Version control dbt projects. You get free audit trails and rollback capability

This pattern has become industry standard for good reason — it separates concerns, enables testing at every layer, and makes teams move faster.
