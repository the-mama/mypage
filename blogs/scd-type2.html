Slowly Changing Dimension Type 2 (SCD Type 2) Implementation

SCD Type 2 is used to maintain historical changes in dimension tables by creating new records instead of overwriting existing ones. This is critical for auditability in data warehouses.

Business Requirement

We needed to track changes in customer address and status over time while preserving historical data.

Design Approach

  • Surrogate Key used as primary key
  • Effective Start Date / End Date columns
  • Current Flag (IsCurrent)

Table Structure

Customer_Dim
-------------
CustomerKey (PK)
CustomerID
Name
Address
StartDate
EndDate
IsCurrent

Logic Flow

IF record exists AND data changed:
    - Expire old record (EndDate = GETDATE(), IsCurrent = 0)
    - Insert new record (IsCurrent = 1)
ELSE
    - Insert new record

SQL Implementation (Simplified)

UPDATE Customer_Dim
SET EndDate = GETDATE(),
    IsCurrent = 0
WHERE CustomerID = @CustomerID
AND IsCurrent = 1;

INSERT INTO Customer_Dim
(CustomerID, Name, Address, StartDate, EndDate, IsCurrent)
VALUES
(@CustomerID, @Name, @Address, GETDATE(), NULL, 1);

Benefits

  • Full history tracking
  • Time-travel analytics
  • Accurate reporting over time

Real Project Usage

This pattern was used in AddressBase Premium data integration to track property changes over time.
